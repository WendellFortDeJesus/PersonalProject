# **App Name**: PatronPoint

## Core Features:

- Visitor Authentication: Enable visitors (Students/Faculty/Staff) to log in using RF-ID or Email for quick and secure access.
- Purpose Selection: Allow visitors to select their purpose for visiting the library from a pre-defined list, enhancing data collection.
- Confirmation Screen: Display a clear confirmation screen to visitors upon successful check-in, providing immediate feedback.
- Admin Secure Login: Provide a secure email and password login system for library staff to access the backend dashboard.
- Metrics and Reporting Dashboard: Allow administrators to view key visitor metrics and generate reports based on check-in data, visit purpose, and time.
- User Access Management: Enable administrators to manage visitor access, including blocking or unblocking specific RF-ID or email users.

## Style Guidelines:

- A light color scheme to convey a professional and clean academic environment.
- Primary color: A deep, professional blue (#355872) for headings and key interactive elements.
- Secondary color: A calming, lighter blue (#7AAACE) for background accents and secondary information.
- Accent color: A vibrant light blue (#9CD5FF) for interactive elements, highlights, and alerts.
- All text will use 'Fumi' and 'Antinomy Studio' for a unique and modern aesthetic, ensuring readability across UI elements and data displays.
- Use clean, clear, and universally recognizable line icons that contribute to a professional and intuitive user experience across both the visitor kiosk and admin dashboard.
- Visitor interface will feature large, clear touch targets and streamlined flows for efficiency. The admin dashboard will use a responsive grid layout to organize metrics, reports, and management tools effectively.
- Incorporate subtle feedback animations for button presses and successful actions, and smooth transitions between screens to enhance the perceived responsiveness and polish of the application.
